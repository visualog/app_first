# 버전 관리 매니저 스킬

이 스킬은 Expo 프로젝트의 `package.json`과 `app.json` 버전을 동기화하여 관리하고, Git 태그 생성을 자동화합니다.

## 사용 가능 명령어

1. **릴리즈 패치**: 패치 버전을 1 올립니다 (예: 0.1.0 -> 0.1.1).
2. **릴리즈 마이너**: 마이너 버전을 1 올리고 패치를 0으로 초기화합니다 (예: 0.1.5 -> 0.2.0).
3. **릴리즈 메이저**: 메이저 버전을 1 올리고 마이너와 패치를 0으로 초기화합니다 (예: 0.2.0 -> 1.0.0).

## 작동 방식

- `mobile/scripts/release.js` 스크립트를 사용하여 `package.json`과 `app.json`의 `version` 필드를 업데이트합니다.
- 변경 사항을 `chore: release vX.X.X` 메시지로 커밋합니다.
- `vX.X.X` 형식으로 Git 태그를 생성합니다.

## 실행 방법

사용자가 "릴리즈 패치"와 같이 입력하면, 에이전트는 `.agent/workflows`에 등록된 워크플로우를 참조하여 작업을 수행합니다.
직접 스크립트를 실행하려면 다음 명령어를 사용하십시오:
```bash
node mobile/scripts/release.js <patch|minor|major>
```
